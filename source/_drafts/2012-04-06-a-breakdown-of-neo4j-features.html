Hi everyone,

We get people interested in Neo4j from all stages of development and types of backgrounds. Neo4j has a lot of features and intricacies, and I thought it would be beneficial to elaborate on what we mean when we discuss Neo4j's nuances.

Schema
Neo4j is a schema-less database. Graphs are naturally flexible and have expressive structures. Structural conventions naturally arise in any given domain, and it is the responsibility of the developer to ensure a high-fidelity transfer of these structures into the graph. Personally I like to use tests to verify these structures are stable over time.

Consistency
Neo4j is a write-consistent database. Neo4j is ACID transactional and write consistency is one of the hallmarks of such systems.

Durability
On transaction commit, all data is flushed to disk. There is no window of opportunity for data to be lost, once a transaction has been accepted it is considered durable since even if the machine crashes, the commit will be correctly recovered from the log.

Decentralized
When running Neo4j in HA mode, a master slave cluster is formed. In this mode the slaves are eventually consistent with respect to the master in the general case (unless you choose to write through a slave, in which case that particular slave and the master will be temporarily eagerly consistent). This configuration allows Neo4j to scale horizontally for writes, however it requires that a full replica of the data-set be present on each machine. This means it's impractical to use Neo4j in this configuration for very large (many terabytes) of data because it's inconvenient to have to replicate so much.

Stability
Neo4j has been in 24x7 production use since 2003. It currently runs mission critical services from small start-ups to blue chip companies like Deutsche Telekom, Adobe, and Cisco.

Complexity of development
Neo4j supports a Java API and a REST API, and in each API a higher level query language called Cypher is available. There are many libraries that will adapt these APIs for your preferred programming language. Furthermore there is a 3rd party stack called Tinkerpop which provides another high-level query language called Gremlin. Which of these stacks you choose depends on your preference.

Security and Privilege
Neo4j can run in-process with your application, in which case it benefits from the security features that you have built in. If you choose to use Neo4j in server mode then the REST API is secured through HTTPs for privacy and has a set of programmable rules to allow or disallow access to different aspects of the server and data.

Reliable, Fault tolerance
Neo4j is classified as as dependable system. It is a transactional store which is fully recoverable. Furthermore the HA cluster provides fail-over in the event of machine failure.
