Hey everyone,

Last week, our Commercial License webinar was designed to discuss Neo4j license options, in order for those considering our advanced and enterprise versions to know what they would be receiving.


Here are the questions asked, with our responses. Thanks everyone for attending!

What happens if I use Neo as a service, for internal use only, and it is not embedded in any other software package?
That's an interesting question. Accessing Neo4j through the built-in Webadmin does not require a license. A web browser communicates with Neo4j over the network, so it might seem that it is exposed to the AGPL requirements. However, a browser is obviously interacting with a generic http service. So this use would not require a license.

It's only when software has been created to interact specifically with Neo4j that integration occurs and licensing considerations come into play. Internal use versus external use does not affect licensing considerations. It's all about whether you have created software that is written to work with Neo4j. Such software should be open source, or acquire a commercial license.

What if I want to use Neo as part of the SaaS services?
Neo4j could be made available directly to end-users as part of a SaaS service, with the licensing obligations passed on to the users themselves rather than the service.
[This requires a bit of thought actually. It could require an OEM license, but I'm not sure why.]

What does "integrated software" mean? If I run it as a standalone product that services API requests by other commercial software. Is the other commercial software "integrated?"
Integrated means writing software that conforms with the protocol of requests and responses exchanged through an interface. If the commercial software only corresponds with generic http or jmx services, then it is not integrated with Neo4j. If the commercial software interacts with Neo4j to perform graph operations for storing or querying data, then it has been integrated.

What are the factors determining the actual price of the license?
The pricing is based on the our estimate of the fair market value for the capabilities of Neo4j, as compared to similar enterprise database systems.

If one buys the commercial license and ships the product, are additional costs involved per use, or is the license a one time cost?
If you are repackaging Neo4j into another product, or shipping it as part of a bundle, then an OEM license is appropriate.
