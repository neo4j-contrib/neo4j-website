Documentation


there now are the beginnings of a one-stop documentation that generates HTML output and PDF documents, giving you the reference documentation on using Neo4j and the related components. Also, this documentation will be versioned, so that for each Neo4j release there will be a corresponding version of the Neo4j Reference Manual. Eventually, we hope to evolve this into a book on Neo4j. If you want to contribute to the effort, let us know!


High Availability


With this milestone, we are proud to for the first time release a version of the Highly Available Neo4j cluster to the community. This component turns Neo4j into a system with a Write-Master and a number of Read-Slaves. Upon mutating operations, at least the master and one slave are committing the transaction before propagating out the changes to the other slaves. In case of the write-master failing, the remaining slaves will automatically elect a new master. Also, the whole cluster infrastructure is transparent to the client code, so that all slaves and masters can be written to and read from. The client code does not need to be aware of the type of HA setup it is running on. Simple do

Server extensions


One of the big possibilities with the Neo4j Server is to extend it. With this release, we are moving on to this by providing a simple way to mount your own REST resources into the server. This gives a number of interesting opportunities. Let's just think of a few that the community could contribute:


  • take a script as a payload from a REST call and execute it on the server side (JavaScript, Gremlin, etc)

  • Accept custom payload and execute it in a transaction on the server, good for instance for custom bulk inserts via REST

  • write your own domain REST API

  • on certain REST URLs, execute server-side scripts or resources, scripted in Java, Scala, Gremlin or JavaScript, effectively creating stored procedures.
